# مدير مصاريف الزواج

تطبيق سطح مكتب لإدارة مصاريف تجهيز الشقة وحفل الزفاف، مبني بـ Python و Tkinter مع قاعدة بيانات SQLite محلية. البرنامج مناسب لتسجيل البنود، التوريدات، التسديدات، السلف، المرفقات، وإخراج تقارير منظمة.

## أهم الميزات

- تسجيل مصاريف الفرش، التشطيب، حفل الزفاف، والمصاريف الأخرى.
- دعم أكثر من نظام حسابات:
  - نظام الطرف الواحد.
  - نظام 50/50.
  - نظام بنود على الطرفين.
- حساب نصيب كل طرف حسب النظام المختار.
- متابعة التوريدات والتسديدات والرصيد.
- إدارة السلف بعملات متعددة ومتابعة المسدد والمتبقي.
- حفظ مرفقات وفواتير لكل بند.
- تصدير البيانات إلى Excel.
- إنشاء تقارير PDF للقوائم عند توفر مكتبات PDF المطلوبة.
- واجهة عربية RTL مع إعدادات أسماء الأطراف والغرف وجهات الدفع.

## التشغيل

1. ثبت Python 3 على Windows.
2. ثبت المتطلبات:

```powershell
py -m pip install -r requirements.txt
```

3. شغل البرنامج بإحدى الطرق:

```powershell
.\launch_windows.bat
```

أو للتشغيل مع نافذة أخطاء:

```powershell
.\launch_debug.bat
```

يمكن تشغيله مباشرة أيضا:

```powershell
py app.py
```

## الملفات

- `app.py`: نسخة التشغيل مع كونسول، مفيدة للتطوير أو تتبع الأخطاء.
- `app.pyw`: نسخة تشغيل بدون كونسول، مستخدمة مع لانشر Windows.
- `launch_windows.bat`: تشغيل سريع للبرنامج.
- `launch_debug.bat`: تشغيل Debug مع إبقاء النافذة مفتوحة عند الخطأ.
- `launch_windows.vbs`: تشغيل صامت من Windows Script Host.
- `assets/icons`: أيقونات بطاقات الملخص داخل الواجهة.
- `app_data`: بيانات محلية يتم إنشاؤها/استخدامها وقت التشغيل ولا يتم رفعها على GitHub.

## البيانات المحلية

قاعدة البيانات موجودة محليا داخل:

```text
app_data/apartment_costs.db
```

هذا الملف غير مرفوع على GitHub حفاظا على الخصوصية. عند أول تشغيل في جهاز جديد سيقوم البرنامج بإنشاء مجلدات البيانات وقاعدة SQLite تلقائيا.

## ملاحظات التطوير

- التطبيق يستخدم SQLite، لذلك لا يحتاج خادم قاعدة بيانات.
- مرفقات الفواتير تحفظ داخل `app_data/invoices`.
- إيصالات السلف تحفظ داخل `app_data/loan_receipts`.
- ملفات التصدير تحفظ داخل `app_data/exports`.
- عند حدوث خطأ في بداية التشغيل يتم تسجيله في `app_data/startup_error.log`.
